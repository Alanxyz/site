Basicamente se usan tres librerias time, re y requests

time sirve para cosas relacionadas al tiempo, lo uso para esperar X segundos antes de algo

re permite usar Regex (Regex es un lenguaje que permite escribir patrones de busqueda), lo use para poder encontrar los tokens

Y finalmente requests, lo use para poder interactuar con la API de GitHub

Defini url que contenia la URL de la API

Y hago un bucle del 1 al 1000 (GitHub te deja buscar de la primera a la pagina numero 1000 usando su API)

Defino total a 0 para usarlo como contador de la cantidad de tokens que se encontraron

Aqui defino response, que es la peticion HTTP (GET) que hago a la API

por cada iteracion cambia la pagina

Reviso si el JSON que devolvio response contiene la key items

Basicamente si devolvio repositorios o no

Aca itero por cada post, defino match a una busqueda usando Regex en la descripcion de cada repositorio.

El regex es este \d+\w{20,26}_\w+

Basicamente revisa por un Digito (O mas) al lado de un string con numeros y letras que dura de 20 a 26 caracteres

Eso encuentra un Token de NGROK exactamente (Siempre habran errores, pero con la busqueda que hice no paso)

Si el numero de matches es mayor a 0 sumo 1 al total de tokens encontrado

Abro un archivo llamado dump.txt y guardo el token en ese archivo

Muestro cuantos tokens se obtuvieron en la pagina y espero otros 60 segundos para repetir

Y finalmente, termine con este dump (Junte algunos que encontre anteriormente en GitHub)
